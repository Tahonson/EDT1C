## 1. Быстрый старт (если вы раньше работали только в Конфигураторе)

Цель: за 1 проход поднять рабочую среду и начать доработки через EDT + Git + Cursor.

### 1.1 Что поставить (минимум)

1. Платформа 1С: `8.3.27+` ([releases.1c.ru](https://releases.1c.ru/)).
2. EDT: ветка `2026.x` ([releases.1c.ru](https://releases.1c.ru/)).
3. Java: JDK, совместимый с выбранной EDT ([bell-sw.com/downloads](https://bell-sw.com/pages/downloads/)).
4. Cursor + расширения для 1С ([cursor.com](https://www.cursor.com/)).

### 1.2 Линейный старт

1. Сделайте резервную копию базы.
2. Закройте Конфигуратор с этой базой.
3. В EDT выполните импорт:
   - `Файл -> Импорт -> 1C:Enterprise -> Конфигурация из информационной базы`.
4. Укажите пользователя с полными правами и корректную версию проекта.
5. Если есть расширения — отметьте их при импорте.
6. Дождитесь построения/индексации EDT.
7. Подключите Git (через EDT UI; терминал — альтернативно).
8. Привяжите ИБ в свойствах проекта EDT и выполните `F7`.
9. Для внешних обработок/отчетов создайте отдельный проект EDT и работайте через него.
10. В Cursor установите 1С-расширения и проверьте поиск по `*.bsl`.

Для существующей базы часто быстрее стартовать так:

1. Откройте окно информационных баз в EDT.
2. Перетащите нужную ИБ в область `Навигатор`.
3. Подтвердите импорт проекта (EDT предложит действия автоматически).

![GIF: импорт базы в проект EDT](docs/gifs/Импорт%20базы%20данных%20в%20проект%20EDT.gif)

### 1.3 Состав рабочей области EDT (обязательный минимум)

Рекомендуемая структура одной рабочей области EDT для одного прикладного решения:

- один проект основной конфигурации;
- один или несколько проектов расширений;
- один проект внешних отчетов/обработок;
- одна привязанная ИБ для запуска и отладки.

Это снижает рассинхрон между исходниками, внешними `.epf/.erf` и тем, что реально запускается в отладке.

Ссылочная база для продолжения: раздел `7. Карта полезных материалов и ссылок`.

## 2. EDT + Напарник

<a id="sec-1-1"></a>
### 2.1 Требования и развертывание

Алгоритм установки рабочей среды:

1. Платформа 1С. Скачайте с [releases.1c.ru](https://releases.1c.ru/) и установите платформу `8.3.27+`.
2. EDT. Скачайте с [releases.1c.ru](https://releases.1c.ru/) актуальную версию ветки `2026.x`.
3. Java. Установите `BellSoft Liberica JDK` (версия — согласно требованиям скачанной EDT).
4. Плагин Напарника. В EDT откройте `Help -> Install New Software`, добавьте репозиторий плагина Напарника и установите плагин.
5. Авторизация. Перезапустите EDT, затем в появившемся окне выполните авторизацию под УЗ портала ИТС.

![GIF: создание проекта EDT](docs/gifs/Создание%20проекта%20EDT.gif)

Важно:
- EDT имеет строгие требования к версии платформы; иначе проект может не открыться/не собраться.
- Первый импорт конфигурации обычно занимает заметно больше времени, чем последующие обновления.
- Для повседневных правок/рефакторинга Cursor обычно быстрее, чем Напарник на длинных задачах.

### 2.2 Использование автопродолжения кода (Автокомплит)

Алгоритм работы:

1. Начните писать код или комментарий к логике.

3. Нажмите `Tab` для принятия всего блока или `Ctrl+Right` (зависит от настроек) для принятия по словам.

<a id="sec-1-3"></a>
### 2.3 Генерация кода по промпту

<details>
<summary>Видео-нарезка: 1С:Напарник (генерация по промпту)</summary>
<a href="https://www.youtube.com/watch?v=V441Ih3T3NE&t=2271s"><img src="https://img.youtube.com/vi/V441Ih3T3NE/hqdefault.jpg" alt="Видео: 1С:Напарник (генерация по промпту)" width="900" /></a>
</details>

Алгоритм генерации:

1. Установите курсор в пустую строку внутри процедуры.
2. Напишите комментарий-инструкцию (промпт).
Пример хорошего промпта:

```bsl
// Сформируй запрос к регистру накопления ОстаткиТоваров. Обороты, условие по Складу (из параметров), сгруппируй по Номенклатуре.
```
Пример плохого промпта:

```bsl
// Получи остатки
```
3. Нажмите `Enter`. Напарник сгенерирует текст ниже.
4. Обязательная проверка: убедитесь, что имена реквизитов/переменных совпадают с реальными метаданными.

Критично для качества результата:

- давайте максимально подробную и ограниченную инструкцию;
- явно указывайте, что нельзя удалять существующие комментарии/бизнес-логику;
- проверяйте diff до применения, потому что Напарник может переписать лишнее.

## 3. EDT + MCP

<a id="sec-2-1"></a>
### 3.1 Установка MCP-плагина в EDT

Алгоритм установки в EDT:

1. Откройте в EDT: `Справка -> Установить новое ПО`.
> [!WARNING]
> TODO (медиа): вставить GIF "Установка MCP-плагина в EDT через `Справка -> Установить новое ПО`".
> Рекомендуемое имя файла: `docs/gifs/Установка MCP плагина EDT.gif`.
2. Добавьте update site из нужного релиза `EDT-MCP`.
3. Источник релизов: [EDT-MCP Releases](https://github.com/DitriXNew/EDT-MCP/releases).
4. Подбирайте релиз строго под версию EDT:
   - для `EDT 2026.1+` — актуальный релиз ветки;
   - для `EDT 2025.x` — используйте совместимый старый релиз (практический ориентир: [v1.26.1](https://github.com/DitriXNew/EDT-MCP/releases/tag/v1.26.1)).
5. Перезапустите EDT и включите MCP-плагин в настройках EDT.

Важно:

- ошибка запуска вида `exit code 1`/ошибки Java часто вызвана не JDK, а несовместимым релизом `EDT-MCP`;
- в первую очередь проверьте совместимость версии плагина и EDT.


![GIF: настройка MCP EDT (включение)](docs/gifs/Настройка%20MCP%20EDT%20(включение).gif)

<a id="sec-2-2"></a>
### 3.2 Настройка MCP в Cursor

Алгоритм подключения в Cursor:

1. Откройте `Cursor Settings -> Features -> MCP`.
![GIF: настройка MCP в Cursor](docs/gifs/настройка%20MCP%20в%20Cursor.gif)
2. Добавьте MCP-серверы и проверьте права (read/write по вашей политике доступа).
3. Протестируйте 2–3 сценария: поиск по коду, навигация по проекту, точечная правка.

Шаблон `mcp.json`:

```json
{ "mcpServers": { "1c-docs": { "url": "https://bulzavr.ru/mcp" }, "EDT MCP Server": { "type": "sse", "url": "http://localhost:8765/mcp" } } }
```

На практике самый удобный сценарий — работать через Cursor/VS Code: там проще держать контекст файлов и видеть diff рядом с изменениями.



<a id="sec-2-3"></a>
### 3.3 Частые проблемы EDT (критично)

Ошибка запуска EDT: `exit code 1` (часто конфликт Java):

1. Найдите файл `1cedt.ini` в папке установки EDT (через ярлык -> `Расположение файла`).
2. Добавьте в начало файла:

```text
-vm
C:\Program Files\Axiom\AxiomJDK-Pro-17-Full\bin\server\jvm.dll
```

3. Укажите путь к реально установленной Java на вашей машине.
4. Перезапустите EDT.

Примечание: если проблема появилась после установки MCP-плагина, сначала проверьте совместимость релиза плагина с вашей EDT.

## 4. EDT + IDE (Cursor / VS Code)

<a id="sec-3-1"></a>
### 4.1 Выбор решения / модели
Настройки моделей (Cursor -> Settings -> Models):

1. `Primary model` (для сложных задач): `Claude, Gemeni, Chat GPT` — для архитектуры и сложных запросов/ задач.
2. `Fast model` (для мелких правок): `можно авто и остальные модели, которые уже есть` — для быстрых итераций. Не рекомендуемый вариант.

Для критичных изменений всегда делайте ручную проверку и прогон сценариев.

<a id="sec-3-2"></a>
### 4.2 Настройка плагинов, MCP, правил (rules)

<details>
<summary>Видео-нарезка: Cursor (MCP/IDE)</summary>
<a href="https://www.youtube.com/watch?v=V441Ih3T3NE&t=2562s"><img src="https://img.youtube.com/vi/V441Ih3T3NE/hqdefault.jpg" alt="Видео: Cursor (MCP/IDE)" width="900" /></a>
</details>

Алгоритм настройки рабочей области:

1. Установите расширения для 1С в Cursor:
   - поддержка BSL (подсветка/навигация),
   - дерево конфигурации 1С,
   - CodeTour (для маршрутов по проекту и онбординга).
2. Быстрая установка расширений (через `Ctrl+P` -> `ext install ...`):
   - `ext install 1c-syntax.language-1c-bsl`
   - `ext install whiterabbit.1c-configuration-tree`
3. Откройте `Cursor Settings -> Features -> MCP`.
4. Подключите минимум 2 MCP:
   - `1c-docs` (документация и справочный контекст),
   - `EDT MCP Server` (контекст проекта EDT).
5. Создайте файл `.cursorrules` в корне проекта.
6. Вставьте базовые правила (пример):

```text
Язык программирования: 1С (BSL).
Использовать только русский синтаксис.

Политика изменений:
- Менять только то, что требуется задачей (без “скрытых” архитектурных рефакторингов).
- Перед любыми изменениями, влияющими на метаданные, сверять имена реквизитов/методов по текущему проекту (через EDT).
- По возможности расширять функциональность через расширения, а не через правку “ядра”.
```

Серая зона (to be continued):
- дополнительные MCP под задачи команды (кроме `1c-docs` и `EDT MCP Server`);
- матрица прав доступа MCP (read-only/read-write) по ролям.

Если доступ к marketplace ограничен (без VPN), используйте установку через VSIX:

1. Скачайте плагины вручную:

<a id="sec-5-5"></a>
### 6.5 Алгоритм обновления базы (Слияние с веткой поставщика)

<details>
<summary>Видео-нарезка: обновление базы (в конфигураторе)</summary>
<a href="https://www.youtube.com/watch?v=V441Ih3T3NE&t=2010s"><img src="https://img.youtube.com/vi/V441Ih3T3NE/hqdefault.jpg" alt="Видео: обновление базы (в конфигураторе)" width="900" /></a>
</details>

Самый критичный процесс. Строго соблюдайте последовательность.

Важно:

- обновление типовой конфигурации поставщика не выполняйте через Cursor;
- для больших обновлений ориентируйтесь на отдельный процесс в EDT/Конфигураторе с выделенной веткой поставщика.

Практический разбор процесса:
- [Обновление типовой конфигурации в EDT на примере + скрипт](https://infostart.ru/1c/articles/2585647/)

Исходные данные: у вас есть ветка `vendor` (чистая конфигурация поставщика) и ветка `master/develop` (ваша доработка).

Алгоритм обновления:

1. Обновление ветки `vendor`:
   - `git checkout vendor`
   - обновите базу в EDT (F5)
   - в EDT: ПКМ на проекте -> `Импорт` -> `1С:Предприятие` -> `Конфигурация из файла`
   - выберите новый `.cf` от 1С и дождитесь окончания разбора
   - `git add .` и `git commit -m "vendor: Обновление типовой конфигурации до релиза X.X.X.X"`
2. Подготовка к слиянию:
   - `git checkout master` (или вашу ветку разработки)
   - в EDT дождитесь перестроения проекта на старую ветку
3. Запуск слияния:
   - в терминале: `git merge vendor`
4. Разрешение конфликтов (Merge Tool):
   - если конфликтов нет — переходите к шагу 6
   - если конфликты есть — в EDT откройте Merge Tool по конфликтному метаданному объекту
   - перенесите нужные блоки кода и реквизиты в центральную панель, сохраните
   - в терминале завершите merge после разрешения конфликтов (при необходимости отдельным commit)
5. Завершение merge-коммита:
   - `git commit -m "merge: Слияние релиза X.X.X.X с доработками"`
6. Обновление базы:
   - в EDT выполните `F7` и проведите стандартное обновление базы данных 1С (при необходимости через F5/запуск предприятия для обработчиков обновления).

<a id="sec-5-6"></a>
### 6.6 Серая зона Git в EDT (to be continued)

Нужно зафиксировать командный стандарт:

- целевая ветка для ежедневной работы (`main`/`develop`) и правило, куда мержим feature;
- формат имен веток и коммитов (обязательный шаблон);
- кто и когда делает merge/rebase, и где это делается (EDT UI/терминал);
- минимальный набор проверок перед merge (чек diff, обновление ИБ, smoke-сценарий).

<a id="team-standard"></a>
## Рекомендуемый командный стандарт (кратко)

- Планировать нетривиальные задачи до генерации кода.
- Давать ИИ ограниченный, но достаточный контекст.
- Работать малыми изменениями и регулярно ревьюить diff.
- Не отключать человеческий контроль: тесты и код-ревью обязательны.
- Поддерживать единые версии инструментов и единый git-процесс по команде.
